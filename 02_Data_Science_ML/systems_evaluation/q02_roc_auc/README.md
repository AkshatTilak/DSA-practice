# ROC AUC (Receiver Operating Characteristic - Area Under the Curve)

## 🗂️ Concept Overview
**ROC AUC** is one of the most critical evaluation metrics for binary classification models. While accuracy can be misleading—especially in the presence of **imbalanced datasets**—ROC AUC provides a threshold-independent measure of a model's ability to distinguish between the positive and negative classes.

The **ROC Curve** is a probability curve that plots the **True Positive Rate (TPR)** against the **False Positive Rate (FPR)** at various classification thresholds. The **AUC (Area Under the Curve)** summarizes this curve into a single number between 0 and 1.

### Core Objective
The primary goal of ROC AUC is to quantify the **separability** of the model. An AUC of 1.0 represents a perfect classifier, while an AUC of 0.5 represents a classifier that performs no better than random guessing.

---

## 📐 Theoretical Foundations & Math

### 1. The Building Blocks
To understand ROC AUC, we must first define the components of the confusion matrix relative to a specific threshold $\tau$:

*   **True Positive Rate (TPR) / Sensitivity / Recall**: The proportion of actual positives correctly identified.
    $$\text{TPR} = \frac{TP}{TP + FN}$$
*   **False Positive Rate (FPR)**: The proportion of actual negatives incorrectly identified as positives.
    $$\text{FPR} = \frac{FP}{FP + TN}$$

### 2. The ROC Curve
The ROC curve is generated by varying the threshold $\tau$ from $1$ down to $0$. For every $\tau$, we calculate the TPR and FPR.
*   **X-axis**: False Positive Rate (FPR)
*   **Y-axis**: True Positive Rate (TPR)

### 3. The AUC Calculation
Mathematically, the AUC is the integral of the ROC curve:
$$\text{AUC} = \int_{0}^{1} \text{TPR}(\text{FPR}) \, d\text{FPR}$$

### 4. Probabilistic Interpretation
A powerful way to think about AUC is through **ranking**. AUC is equivalent to the probability that the model will rank a randomly chosen positive instance higher than a randomly chosen negative instance:
$$P(\text{Score}(\text{positive}) > \text{Score}(\text{negative}))$$

## 🛠️ Step-by-Step Logic

To approximate the AUC programmatically without using high-level libraries like `scikit-learn`, we follow the **Trapezoidal Rule** approach.

### Algorithmic Implementation
1.  **Pair and Sort**: Combine the predicted probabilities and the true labels into pairs. Sort these pairs in **descending order** based on the predicted probability.
2.  **Initialize Counters**: Keep track of the total number of positive samples ($P$) and negative samples ($N$) in the dataset.
3.  **Iterate and Accumulate**: Traverse the sorted list. When you encounter a positive sample, you increase the "height" of the curve (TPR). When you encounter a negative sample, you move the "width" of the curve (FPR).
4.  **Trapezoidal Integration**: As you move along the X-axis (FPR), the area added is a rectangle (or trapezoid) formed by the current TPR and the step in FPR.
5.  **Final Calculation**: The sum of these areas divided by $(P \times N)$ gives the AUC.

### Optimal Code Implementation

```python
def solve_optimal(y_true, y_prob):
    """
    Calculates the Area Under the ROC Curve (AUC) using the trapezoidal rule.
    
    Args:
        y_true (list): Ground truth binary labels (0 or 1).
        y_prob (list): Predicted probabilities for the positive class.
    Returns:
        float: The calculated AUC.
    """
    # 1. Sort predictions and labels by probability in descending order
    data = sorted(zip(y_prob, y_true), key=lambda x: x[0], reverse=True)
    
    tp = 0  # True Positives count
    fp = 0  # False Positives count
    last_fp = 0
    last_tp = 0
    auc = 0.0
    
    # Count total positives and negatives
    total_pos = sum(y_true)
    total_neg = len(y_true) - total_pos
    
    if total_pos == 0 or total_neg == 0:
        return 0.0 # Handle edge cases with single-class data

    for i in range(len(data)):
        prob, label = data[i]
        
        if label == 1:
            tp += 1
        else:
            fp += 1
        
        # When we encounter a negative sample (or reach the end), 
        # we calculate the area of the trapezoid formed since the last FPR change.
        if prob != data[i+1][0] if i+1 < len(data) else True:
            # Trapezoid Area = (width) * (average height)
            # width = (fp - last_fp) / total_neg
            # height = (tp + last_tp) / (2 * total_pos)
            auc += ((fp - last_fp) / total_neg) * ((tp + last_tp) / (2 * total_pos))
            last_fp = fp
            last_tp = tp
            
    return auc
```

## ⚙️ Complexity & Training Details

### Computational Complexity

| Phase | Time Complexity | Space Complexity | Reason |
| :--- | :--- | :--- | :--- |
| **Sorting** | $O(N \log N)$ | $O(N)$ | Sorting predictions is the dominant factor. |
| **Traversal** | $O(N)$ | $O(1)$ | Single pass through the sorted list. |
| **Total** | **$O(N \log N)$** | **$O(N)$** | Due to the necessity of sorting for ranking. |

### Key Considerations
*   **Tied Probabilities**: If two samples have the same predicted probability but different labels, the ROC curve moves diagonally. The trapezoidal rule handles this by calculating the area only after all samples with the same probability are processed.
*   **Threshold Independence**: Unlike Accuracy or F1-Score, AUC does not require you to pick a cutoff (e.g., 0.5). It evaluates the model's performance across **all possible** cutoffs.

## 🌍 Real-World Applications

### 1. Medical Diagnosis
In cancer detection, a **False Negative** (missing a tumor) is far more dangerous than a **False Positive** (extra tests). ROC AUC allows doctors to choose a threshold that maximizes Sensitivity (TPR) while keeping the False Positive Rate acceptable.

### 2. Fraud Detection
Fraud datasets are typically highly imbalanced (e.g., 99.9% legitimate, 0.1% fraud). An accuracy of 99.9% is trivial but useless. ROC AUC helps engineers evaluate if the model can actually push the few fraud cases to the top of the probability rankings.

### 3. Search Engine Ranking
The core of search ranking is retrieving relevant documents before irrelevant ones. Since the underlying mechanism is ranking, AUC is a direct proxy for how well the ranking algorithm separates relevant content from noise.

### 4. Credit Scoring
Banks use AUC to determine the probability of default. By analyzing the ROC curve, they can decide the exact probability threshold above which a loan application should be rejected to balance risk vs. profit.
