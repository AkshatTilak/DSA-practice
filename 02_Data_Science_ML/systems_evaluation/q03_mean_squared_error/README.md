# Mean Squared Error (MSE)

## 🗂️ Concept Card: Systems Evaluation
**Mean Squared Error (MSE)** is a fundamental risk function and evaluation metric used primarily in **regression analysis** to measure the average squared difference between the estimated values ($\hat{y}$) and the actual value ($y$). It serves as a proxy for the "cost" or "loss" of a model's predictions.

### Core Objective
The primary goal of MSE is to quantify the **variance** and **bias** of a model. By squaring the errors, MSE ensures that large errors are penalized more heavily than small errors, making it an ideal loss function for optimization algorithms like **Gradient Descent**, as it provides a smooth, differentiable surface.

---

## 📐 Theoretical Foundations & Math

### The Mathematical Formula
The Mean Squared Error is defined by the following equation:

$$\text{MSE} = \frac{1}{n} \sum_{i=1}^{n} (y_i - \hat{y}_i)^2$$

**Where:**
- $n$: The total number of observations/samples in the dataset.
- $\sum_{i=1}^{n}$: The summation symbol, indicating we sum the results from the first observation to the $n$-th.
- $y_i$: The **actual ground-truth value** (the target).
- $\hat{y}_i$: The **predicted value** generated by the model.
- $(y_i - \hat{y}_i)^2$: The **squared error** for a single data point.

### Why Square the Error?
1. **Elimination of Negative Signs**: Errors can be positive (overestimation) or negative (underestimation). If we simply summed the errors, they would cancel each other out, potentially yielding an MSE of 0 for a highly inaccurate model.
2. **Sensitivity to Outliers**: Because the error is squared, a prediction that is off by $10$ units contributes $100$ to the sum, whereas a prediction off by $1$ unit contributes only $1$. This forces the model to prioritize reducing large errors.
3. **Mathematical Convergence**: The squared function is differentiable everywhere, which is a requirement for calculating gradients in backpropagation and optimization.

## ⚙️ Step-by-Step Logic

To implement an MSE calculator, the algorithm follows a linear transformation process:

1. **Input Alignment**: Ensure that the list of actual values ($y$) and predicted values ($\hat{y}$) are of the same length. If they differ, the calculation is mathematically undefined.
2. **Calculate Residuals**: For every pair $(y_i, \hat{y}_i)$, calculate the difference: $\text{Error} = y_i - \hat{y}_i$.
3. **Square the Residuals**: Square each individual error: $\text{Squared Error} = (y_i - \hat{y}_i)^2$.
4. **Aggregate**: Sum all the squared errors together.
5. **Normalize**: Divide the total sum by the number of samples ($n$) to find the mean.

### Optimal Implementation

```python
import numpy as np

def solve_optimal(actual, predicted):
    """
    Calculates the Mean Squared Error between two lists/arrays.
    
    Args:
        actual (list/np.array): Ground truth target values.
        predicted (list/np.array): Model predictions.
        
    Returns:
        float: The calculated MSE.
    """
    # Convert to numpy arrays for vectorized operations (Performance Optimization)
    actual = np.array(actual)
    predicted = np.array(predicted)
    
    # 1. Subtract predicted from actual (Element-wise)
    # 2. Square the result (Element-wise)
    # 3. Calculate the mean of those squares
    mse = np.mean((actual - predicted)**2)
    
    return float(mse)

# Example usage
y_true = [3, -0.5, 2, 7]
y_pred = [2.5, 0.0, 2, 8]
print(f"MSE: {solve_optimal(y_true, y_pred)}") # Output: 0.375
```

## 📊 Complexity & Training Details

### Computational Complexity

| Phase | Time Complexity | Space Complexity | Reason |
| :--- | :--- | :--- | :--- |
| **Calculation** | $O(n)$ | $O(1)$ or $O(n)$ | We must visit every element in the dataset once. Space is $O(1)$ if using a loop, or $O(n)$ if creating intermediate numpy arrays. |
| **Inference** | N/A | N/A | MSE is an evaluation metric, not a model; it is computed after inference. |

### Comparison with Other Metrics

| Metric | Formula | Sensitivity to Outliers | Scale |
| :--- | :--- | :--- | :--- |
| **MSE** | $\frac{1}{n}\sum(y-\hat{y})^2$ | High (Strong penalty) | Squared units of $y$ |
| **MAE** | $\frac{1}{n}\sum|y-\hat{y}|$ | Low (Linear penalty) | Same units as $y$ |
| **RMSE** | $\sqrt{\text{MSE}}$ | Moderate | Same units as $y$ |

## 🚀 Real-World Applications

MSE is the standard loss function for various regression-based systems:

1. **Real Estate Pricing**: Predicting house prices based on square footage and location. A large error in a luxury home price is penalized heavily to ensure the model doesn't diverge wildly.
2. **Stock Market Forecasting**: Predicting the closing price of a stock. MSE helps quantify the average volatility of the prediction error.
3. **Temperature Sensors**: In IoT systems, MSE is used to calibrate sensors by comparing their readings against a highly accurate reference thermometer.
4. **Neural Network Training**: In the output layer of a regression neural network, MSE is typically the objective function minimized via **Stochastic Gradient Descent (SGD)**.
